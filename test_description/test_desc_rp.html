2. Test description syntax

The test description files are JSON documents. Each of these documents contains a JSON object with a set of keys and values. The possible keys are:

  • behavior
  • capabilities
  • claims
  • detailed_description
  • expected_result
  • group
  • reference
  • short_description

These will be described one by one below.

2.1. behavior

This is a list of identifiers. Each of them used in the code to decide on actions. This is exclusively used to make responses or requests not standard compliant.

2.2. capabilities

This is used to decided what features the test tool (an OP) should support. Most commonly this is about which response_types that are supported.

2.3. claims

Denotes whether claims returned should only be normal claims or aggregated and/or distributed.

2.4. detailed_description

A description aimied at the user of the test suite.

2.5. expected_result

A description of the expected result.

2.6. group

Which group of tests the test belongs to. As of now the complete set is:

  • 3rd-Party Init SSO
  • Claims Request Parameter
  • Claims Types
  • Client Authentication
  • Discovery
  • Dynamic Client Registration
  • ID Token
  • Key Rotation
  • RP Initiated BackChannel Logout
  • RP Initiated FrontChannel Logout
  • RP Initiated Logout
  • Response Type and Response Mode
  • UserInfo Endpoint
  • nonce Request Parameter
  • request_uri Request Parameter
  • scope Request Parameter

2.7. reference

A link to the part of the standard that is being tested..

2.8. short_description

Short description of what the test tries to test.
